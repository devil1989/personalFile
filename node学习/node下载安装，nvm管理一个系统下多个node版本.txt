nvm下载地址：https://github.com/coreybutler/nvm-windows/releases

nvm on： 启用node.js版本管理
nvm off： 禁用node.js版本管理(不卸载任何东西)
nvm uninstall <version>： 卸载指定版本的nodejs。
nvm ls   // 查看目前已经安装的版本

nvm install 6.10.0  // 安装指定的版本的nodejs
nvm install 8.9.1  // 安装指定的版本的nodejs

nvm use 6.10.0  // 使用指定版本的nodejs，切换对应的node版本到6.10.0


目前公司服务器最新版本是8.9.1，也可以用6.10.0



//安装nvm过程中，会出现各种问题，罗列下：

1.安装的时候，用的是默认的nvm和node安装路径，别去改动，容易出问题

2.安装好以后，在用nvm install 安装node版本的时候，容易出现node或者nvm安装失败，需要在setting文件内添加镜像地址，从淘宝镜像那边下载
  node_mirror: https://npm.taobao.org/mirrors/node/
  npm_mirror: https://npm.taobao.org/mirrors/npm/

3.添加好以后，可能安装高版本的node没啥问题，但是安装8.0以下的低版本node，例如6.10.0，就容易出现安装失败，所以可以直接从node那边下载压缩包，解压出文件夹【非exe安装包】
  然后把文件夹直接放到nvm的安装路径下：例如我的安装路径：C:\Users\ThinkPad\AppData\Roaming\nvm，文件夹放到这里以后改名位v6.10.0;这样nvm use 6.10.0就会切换到这个版本的node了
  node各个版本下载路径：https://nodejs.org/dist/v6.10.0/

4.最最坑的问题，你会发现node ls可以看到安装好的node版本，但是就是没办法切换，从安装路径C:\Users\ThinkPad\AppData\Roaming\nvm去看各个node版本里面的文件基本都是全的
   这个问题困扰我太久了，最后发现，这个tm是因为我打开的git命令行窗口不是管理员权限；在以管理员全选运行cmd，然后执行nvm use来切换node，就可以成功切换了；最最肯爹


5.非常坑的一个问题：nvm安装了某个node版本，比如是8.9.1，但是安装对应的npm失败了，它会提示npm的下载地址，去哪个地址下载对应的npm
                               
		1.解压文件后，改名为npm，把他复制到C:\Users\ThinkPad\AppData\Roaming\nvm\v8.9.1\node_modules 里面，
		就是在C:\Users\ThinkPad\AppData\Roaming\nvm\v8.9.1\node_modules 里面添加npm文件
		
		2.复制C:\Users\ThinkPad\AppData\Roaming\nvm\v8.9.1\node_modules\npm\bin文件夹下的所有文件，拷贝到C:\Users\ThinkPad\AppData\Roaming\nvm\v8.9.1文件夹里面
		  这样就可以使用npm了
